Three tiers, three jobs.

Model size and reasoning effort are separate choices. Start with the work, then decide how much time and compute it deserves.

Sol

Flagship tier. The broadest capability headroom in the family.

Use it when the cost of a weak result is higher than the cost of more compute.

Terra

Balanced tier. A practical middle ground for everyday work.

Use it for repeatable tasks where cost and capability both matter.

Luna

Fast, affordable tier. Built for lower-cost throughput.

Use it for scale, first passes, and work that can trade time or peak quality for savings.

How this guide decides.

A small evidence set is more useful than false completeness. Every recommendation stays inside one benchmark and shows its nearest trade-offs.

Recommendation rule

The guide compares score, estimated benchmark-run cost, and latency. It chooses an editorial start point that avoids a large resource jump for a small score gain. This is not a universal optimization formula.

Benchmark limits

Scores are compared only within the same benchmark. A benchmark resembles a task; it does not reproduce every real workflow, tool, prompt, or failure mode.

Missing-value policy

Missing data remains missing. The project does not fill absent values with zero, interpolate unpublished reasoning levels, or turn source conflicts into averages.

Sources and boundaries

This page embeds a selected subset of the pre-existing schema 0.3.0 datasets. Displayed costs are estimated full benchmark-run costs, not per-task API quotes.
